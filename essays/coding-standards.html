When I work alone, my code will often turn into a big mess of complicated jargon. This is often because I’m working in the pursuit of short term functionality without realizing the problems I’m creating in the future. Writing code neatly is an important and complementary skill to writing functional code. It makes programs more readable and easier to work with. Thus, for the long term health of any project, it’s always good to write neat code. When working in VSCode, I was able to use the ESLint extension for the first time. ESLint acts to standardize code into a specific and readable format. Anytime code is written in a way that doesn’t align with the ESLint formatting, the plugin will mark the section as an error and will often give feedback as to how to write it properly.
